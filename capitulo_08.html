8.1

¿Por Qué Necesitamos Funciones?

Hasta ahora todo el código vive en main(). A medida que los programas crecen, esto produce código espagueti: difícil de leer, de depurar y de reutilizar. Las funciones son la solución.

Sin funcionesCon funciones
main() con 200+ líneasmain() orquesta 8 funciones de 20 líneas cada una
El mismo cálculo copiado 5 vecesUna función calcularIVA() llamada 5 veces
Cambiar una regla requiere buscarla en todo el códigoCambiar la regla en un solo lugar
Probar una parte requiere ejecutar todoCada función se puede probar independientemente
Definición formal

Una función es un bloque de código con nombre, que recibe parámetros (datos de entrada), ejecuta una tarea específica y opcionalmente retorna un valor. Sigue el principio de responsabilidad única: hace una sola cosa bien.

8.2

Anatomía de una Función

C++ · anatomía completa
anatomia.cpp
// ─── TIPO RETORNO ─── NOMBRE ────── PARÁMETROS ──────────────
double              calcularDescuento(double precio, double porcentaje) {
    // CUERPO
    double descuento = precio * (porcentaje / 100.0);
    return descuento;   // RETORNO — valor que regresa al llamador
}

int main() {
    double precio = 150.0;
    double desc   = calcularDescuento(precio, 20.0); // LLAMADA
    cout << "Descuento: $" << desc << endl;    // Descuento: $30
    return 0;
}
ComponenteDescripciónEjemplo
Tipo de retornoTipo del valor que devuelve. void si no devuelve nadadouble
NombreIdentificador descriptivo (verbo + sustantivo)calcularDescuento
ParámetrosLista de datos de entrada (tipo + nombre)double precio, double porcentaje
CuerpoInstrucciones entre llavesEl bloque { ... }
returnValor que se entrega al llamador. Obligatorio si no es voidreturn descuento;
8.3

Funciones void vs Funciones con Retorno

void — acción sin valor de retorno
void mostrarBienvenida(string nombre) {
    cout << "Bienvenido, " << nombre << endl;
    // No hay return
}

// llamada:
mostrarBienvenida("Ana"); // no asignas su resultado
Con retorno — calcula y entrega un valor
double calcularIVA(double monto) {
    return monto * 0.12;
}

// llamada:
double iva = calcularIVA(100.0); // iva = 12.0
📌 Regla práctica

Usa void para funciones que hacen algo (mostrar, imprimir, dibujar). Usa tipo de retorno para funciones que calculan o evalúan algo (calcular, verificar, convertir). Una función que hace ambas cosas probablemente debería dividirse en dos.

8.4

Prototipos — Declarar antes de Definir

C++ lee el código de arriba hacia abajo. Si llamas una función antes de definirla, el compilador no la conoce y genera error. Los prototipos declaran la firma de la función antes de su definición, permitiendo organizarla después de main().

C++ · prototipos
prototipos.cpp
#include <iostream>
using namespace std;

// ─── PROTOTIPOS (declaraciones) ───────────────────────────────
double calcularArea(double base, double altura);
void   mostrarResultado(double area);

int main() {
    double a = calcularArea(5.0, 3.0);
    mostrarResultado(a);
    return 0;
}

// ─── DEFINICIONES (implementaciones) ─────────────────────────
double calcularArea(double base, double altura) {
    return (base * altura) / 2.0;
}

void mostrarResultado(double area) {
    cout << "Área del triángulo: " << area << endl;
}
📌 Prototipo = firma sin cuerpo

El prototipo incluye tipo de retorno, nombre y tipos de parámetros, terminando en punto y coma. Los nombres de parámetros son opcionales en el prototipo, pero conviene incluirlos para documentar mejor.

8.5

Alcance de Variables: Local vs Global

Las variables declaradas dentro de una función son locales: solo existen mientras esa función ejecuta. Las declaradas fuera de todas las funciones son globales: visibles en todo el programa.

⚠️ Evita variables globales como regla de estilo

Las variables globales hacen que las funciones dependan de estado externo, lo que dificulta depurar, probar y reutilizar el código. Prefiere pasar datos como parámetros y recibir resultados con return. Las únicas excepciones razonables son constantes globales (const).

C++ · alcance local vs global
alcance.cpp
const double PI = 3.14159; // constante global: aceptable

int contador = 0; // variable global mutable: EVITAR

double areaCirculo(double radio) {
    double area = PI * radio * radio; // local — solo existe aquí
    return area;
    // 'area' deja de existir al salir de la función
}

int main() {
    double r = 5.0;         // local a main
    cout << areaCirculo(r); // 78.539...
    // 'area' de areaCirculo no es accesible aquí
    return 0;
}
8.6

Diseño Modular — Dividir para Conquistar

Un programa bien diseñado con funciones tiene un main() que es básicamente un guión de director: orquesta las funciones sin ejecutar lógica de negocio directamente.

C++ · diseño modular · calculadora IMC
imc_modular.cpp
#include <iostream>
using namespace std;

// Prototipos
double calcularIMC(double peso, double talla);
string clasificarIMC(double imc);
void   mostrarReporte(double imc, string clasificacion);

int main() {
    double peso, talla;
    cout << "Peso (kg): "; cin >> peso;
    cout << "Talla (m): "; cin >> talla;

    double imc   = calcularIMC(peso, talla);
    string categ = clasificarIMC(imc);
    mostrarReporte(imc, categ);
    return 0;
}

double calcularIMC(double peso, double talla) {
    return peso / (talla * talla);
}

string clasificarIMC(double imc) {
    if      (imc < 18.5) return "Bajo peso";
    else if (imc < 25.0) return "Normal";
    else if (imc < 30.0) return "Sobrepeso";
    else                  return "Obesidad";
}

void mostrarReporte(double imc, string categ) {
    cout << "IMC: " << imc << " → " << categ << endl;
}
🤖 La IA divide bien, pero no siempre con criterio

La IA es buena para dividir código en funciones cuando se lo pides. Sin embargo, suele crear funciones demasiado pequeñas (una línea) o demasiado grandes (todo el main). El criterio es: una función = una responsabilidad clara = un nombre que lo exprese. Si no puedes nombrar la función con un verbo descriptivo, probablemente no debería existir como función separada.
