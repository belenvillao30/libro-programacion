🔬 Unidad III · Sesión Práctica 10

Calculadora de Matemática Discreta

recursión · factorial · potencia · MCD · palíndromo · árbol de llamadas · límites de la pila

Duración180 min (3h)
ModalidadIndividual → Parejas
EntregableCódigo .cpp + trazas + bitácora
F1 · Trazas papel F2 · Pseudocódigo F3 · C++ Individual F4 · Extensión F5 · Análisis IA F6 · Bitácora
Objetivo de la Sesión
El estudiante implementa un sistema de calculadora matemática con funciones recursivas para factorial, potencia, MCD y verificación de palíndromo. Dibuja la traza de pila de cada función antes de codificar, compara con la versión iterativa equivalente y documenta las limitaciones de la recursión en términos de profundidad de pila.
📐 Contexto del Problema
La cátedra de Matemáticas Discretas de ESPAM MFL necesita una calculadora de consola con menú. Operaciones: 1) Factorial n! — 2) Potencia baseⁿ — 3) MCD(a, b) por algoritmo de Euclides — 4) Verificar palíndromo — 5) Suma de dígitos — 6) Salir. Cada operación se implementa como función recursiva. main() solo gestiona el menú y la lectura de datos.
F1
Trazas de Pila en Papel — Antes de Codificar
0 – 25 min · SIN computadora

Las trazas de pila son el equivalente de la simulación manual de bucles. Para cada función recursiva, dibuja la expansión y la contracción de la pila de llamadas.

1
Traza de factorial(5)
Dibuja en papel cada llamada con su valor de n, el cálculo que realiza y el valor que retorna. Usa el formato de la tabla del §10.3. Responde: ¿cuántos marcos de pila hay activos al mismo tiempo en el momento de mayor profundidad? ¿Qué pasaría con factorial(100000)?
2
Traza de mcd(48, 18)
El algoritmo de Euclides: mcd(a, b) → caso base b==0 retorna a; caso recursivo mcd(b, a%b). Traza completa: mcd(48,18) → mcd(18,12) → mcd(12,6) → mcd(6,0) → retorna 6. Verifica cada paso con la operación módulo. ¿Cuántas llamadas recursivas necesita?
3
Identificar caso base y recursivo
Para cada una de las cinco funciones de la calculadora, escribe en papel: (a) la condición del caso base, (b) el valor que retorna en el caso base, (c) la expresión del caso recursivo, (d) cómo cada llamada recursiva se acerca al caso base. Si no puedes responder (d), la función puede ser infinita.
F2
Pseudocódigo en PSeInt
25 – 55 min
PSeInt · factorial recursivo — verificar con traza del F1
Funcion resultado <- factorial(n) Si n = 0 Entonces resultado <- 1 // caso base SiNo resultado <- n * factorial(n - 1) // caso recursivo FinSi FinFuncion
PSeInt · MCD de Euclides recursivo
Funcion resultado <- mcd(a, b) Si b = 0 Entonces resultado <- a // caso base: b=0 → a es el MCD SiNo resultado <- mcd(b, a MOD b) // caso recursivo FinSi FinFuncion
PSeInt · palíndromo recursivo con índices
Funcion resultado <- esPalindromo(s, ini, fin) Si ini >= fin Entonces resultado <- Verdadero // caso base: cadena de 0 o 1 caracteres SiNo Si s[ini] <> s[fin] Entonces resultado <- Falso // caracteres distintos: no es palíndromo SiNo resultado <- esPalindromo(s, ini+1, fin-1) // avanzar hacia el centro FinSi FinFuncion
⚙️ Verificar en PSeInt con los valores del F1

Corre cada función con los valores que trazaste en papel. Si PSeInt da el mismo resultado que tu traza manual, el algoritmo es correcto. Si difieren, hay un error en el pseudocódigo — corrígelo aquí antes de traducir a C++.

Implementación Completa en C++ — Individual
55 – 105 min
C++ · Calculadora de Matemática Discreta — implementación completa
calculadora_discreta.cpp 
#include <iostream>
#include <string>
using namespace std;

// ─── PROTOTIPOS ─────────────────────────────────────────────────
long long factorial(int n);
double    potencia(double base, int exp);
int       mcd(int a, int b);
bool      esPalindromo(const string& s, int ini, int fin);
int       sumaDigitos(int n);
void      mostrarMenu();

// ─── MAIN ────────────────────────────────────────────────────────
int main() {
    int opcion;
    do {
        mostrarMenu();
        cin >> opcion;
        switch (opcion) {
            case 1: {
                int n;
                do { cout<<"  n (0-20): "; cin>>n; } while(n<0||n>20);
                cout << "  " << n << "! = " << factorial(n) << "
";
                break; }
            case 2: {
                double base; int exp;
                cout<<"  base: "; cin>>base;
                do { cout<<"  exp (0-30): "; cin>>exp; } while(exp<0||exp>30);
                cout << "  " << base << "^" << exp << " = " << potencia(base,exp) << "
";
                break; }
            case 3: {
                int a, b;
                cout<<"  a: "; cin>>a;
                cout<<"  b: "; cin>>b;
                cout << "  MCD("<<a<<","<<b<<") = " << mcd(a,b) << "
";
                break; }
            case 4: {
                string s;
                cout<<"  Cadena: "; cin>>s;
                bool res = esPalindromo(s, 0, s.length()-1);
                cout << "  ""<<s<<"" " << (res ? "ES" : "NO ES") << " palíndromo
";
                break; }
            case 5: {
                int n;
                do { cout<<"  Número positivo: "; cin>>n; } while(n<0);
                cout << "  Suma de dígitos = " << sumaDigitos(n) << "
";
                break; }
            case 6: cout<<"Hasta luego.
"; break;
            default: cout<<"Opción inválida.
";
        }
    } while (opcion != 6);
    return 0;
}

long long factorial(int n) {
    if (n == 0) return 1;         // caso base
    return n * factorial(n - 1);   // caso recursivo
}

double potencia(double base, int exp) {
    if (exp == 0) return 1;       // caso base: cualquier número^0 = 1
    return base * potencia(base, exp - 1);
}

int mcd(int a, int b) {
    if (b == 0) return a;         // caso base: b=0 → a es el MCD
    return mcd(b, a % b);         // caso recursivo: Euclides
}

bool esPalindromo(const string& s, int ini, int fin) {
    if (ini >= fin) return true;  // caso base: 0 o 1 caracteres restantes
    if (s[ini] != s[fin]) return false;   // extremos distintos → no palíndromo
    return esPalindromo(s, ini + 1, fin - 1);  // avanzar al interior
}

int sumaDigitos(int n) {
    if (n < 10) return n;          // caso base: un solo dígito
    return (n % 10) + sumaDigitos(n / 10);  // último dígito + resto
}

🤖 Protocolo de Análisis IA · Sesión 10

Pide a la IA que implemente Fibonacci de tres formas: (1) recursiva naïve, (2) recursiva con memoización usando un array, (3) iterativa. Luego responde por escrito: (A) ¿Qué problema resuelve la memoización que no resuelve la recursión simple? (B) Para n=35, ¿cuántas veces se llama fibonacci(2) en la versión naïve vs. memoizada? (C) ¿La memoización añade complejidad al código? ¿Vale la pena para valores pequeños de n? (D) ¿Qué versión agregarías al menú de la calculadora para el caso de uso real?

📌 Límite de uso de IA en esta práctica

Puedes usar la IA para obtener las tres versiones de Fibonacci para el análisis del F5. El resto del código de la calculadora (F3 y F4) debe ser implementación propia. Todo el trabajo entregado debe poder ser explicado en revisión oral.
