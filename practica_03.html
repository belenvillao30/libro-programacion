🎯 Misión de la sesión
Eres el auditor de calidad de un equipo de desarrollo. Tu tarea: recibir tres programas con distintos tipos de errores, diagnosticarlos, corregirlos y elevar su calidad con buenas prácticas. Al final habrás integrado todas las habilidades de la Unidad I: algoritmos → tipos → código limpio y depurado.

Programa A — Errores de sintaxis — Individual
⏱ 20 – 55 minutos

El siguiente programa calcula el área de un triángulo. Tiene 4 errores de sintaxis. Encuéntralos primero en papel, luego verifica con el compilador.

C++ · PROGRAMA A — con 4 errores de sintaxis
programa_a.cpp 
#include <iostream>
using namespace std

int main() {
    double base, altura area;
    cout << "Ingrese la base: ";
    cin >> base;
    cout << "Ingrese la altura: "
    cin >> altura;
    area = (base * altura) / 2;
    cout << "Área = " << area << endl;
    return 0
}

F3
Programa B — Errores lógicos — Individual
⏱ 55 – 100 minutos

Este programa compila sin errores pero produce resultados incorrectos. Contiene 3 errores lógicos. Usa la tabla de simulación para encontrarlos sin ejecutar primero.

C++ · PROGRAMA B — compila pero tiene 3 errores lógicos
programa_b.cpp 
#include <iostream>
using namespace std;

int main() {
    // Calcular promedio ponderado y sueldo neto
    int    nota1 = 8, nota2 = 7, nota3 = 9;
    int    promedio = nota1 + nota2 + nota3 / 3;  // Error 1

    double sueldo = 1200;
    double descuento = 9 / 100;                  // Error 2
    double neto = sueldo - sueldo * descuento;

    double celsius = 100;
    double fahrenheit = celsius * 9 / 5 + 32;     // Error 3 ← analizar
    // Pista: ¿qué pasa si celsius fuera un int?

    cout << "Promedio: "    << promedio    << endl;
    cout << "Sueldo neto: $" << neto        << endl;
    cout << "Fahrenheit: "  << fahrenheit  << endl;
    return 0;
}

✅ Salida correcta (objetivo)

💡 Pista sobre el Error 3

La fórmula celsius * 9 / 5 + 32 puede dar 212 con double celsius = 100, pero fallará si celsius fuera int con valores como 37. El error latente está en la falta de un cast explícito que garantice la división real en todos los casos.

Programa C — Refactorizar con buenas prácticas — Parejas
⏱ 100 – 135 minutos

El Programa C funciona correctamente — compila y da resultados correctos. Pero tiene pésimas prácticas. Tu misión: reescribirlo completo respetando el checklist del §3.5. No cambies la lógica, solo la forma.

C++ · PROGRAMA C — funciona, pero malas prácticas
programa_c_original.cpp 
#include<iostream>
#include<iomanip>
using namespace std;
int main(){
double a,b,c,d,e,f;
cout<<"ingrese dato1: ";cin>>a;
cout<<"ingrese dato2: ";cin>>b;
cout<<"ingrese dato3: ";cin>>c;
b=a*0.12;
c=a*0.095;
d=a-b-c;
e=d*12;cout<<fixed<<setprecision(2);

cout<<"res2: "<<c<<endl;
cout<<"res3: "<<d<<endl;
cout<<"res4: "<<e<<endl;

1
Descifrar el programa original
Antes de reescribir, entiende qué hace. Prueba con: dato1 = 1500. ¿Qué representan b, c, d, e? Pista: piensa en un rol de pagos. Una vez que lo entiendas, ponle nombres descriptivos a todas las variables.
2
Reescribir con nombre de archivo: programa_c_mejorado.cpp
Aplica: encabezado de programa, nombres significativos, constantes const, indentación 4 espacios, comentarios de sección (entrada/proceso/salida), una instrucción por línea, etiquetas descriptivas en el cout.
✅ Verificar antes de pasar a la siguiente fase
Encabezado de programa Nombre del programa, descripción, autor, fecha.
Cero variables de una letra Todas tienen nombre significativo.
Porcentajes como const double No hay literales 0.12 ni 0.095 sueltos.
Indentación consistente 4 espacios, llaves alineadas.
Etiquetas descriptivas El cout dice qué es cada valor (no "res1", "res2").
Salida igual al original Los números calculados son idénticos.
