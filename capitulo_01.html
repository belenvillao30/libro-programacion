1.1

Concepto Formal de Algoritmo

🎥 Explicación visual del concepto

Antes de escribir una sola línea de código es necesario comprender qué es lo que el código ejecuta. Un programa no es otra cosa que la traducción de un algoritmo a un lenguaje que la máquina puede interpretar.

Definición formal

Un algoritmo es una secuencia finita, ordenada y no ambigua de pasos bien definidos que describe cómo resolver un problema. Los algoritmos son independientes del lenguaje de programación: un mismo algoritmo puede implementarse en C++, Python, Java o cualquier otro lenguaje.

Esta independencia del lenguaje es fundamental: la habilidad de diseñar algoritmos correctos es más valiosa —y más transferible— que conocer la sintaxis de un lenguaje específico.

🤖 Reflexión sobre IA

Cuando le pides a una IA que resuelva un problema de programación, ella genera código, no un algoritmo razonado. Si no comprendes el algoritmo subyacente, no puedes verificar si la solución es correcta o eficiente. El algoritmo es la capa que debes dominar tú.

1.2

Características y Propiedades

No toda secuencia de pasos constituye un algoritmo válido. Para serlo, debe cumplir cinco propiedades:

🔢

Finitud

El algoritmo debe terminar después de un número finito de pasos. Un proceso indefinido no es un algoritmo.

📌

Precisión

Cada paso debe estar definido sin ambigüedades. "Dividir la suma entre la cantidad" es preciso; "calcular algo con los datos" no lo es.

📥

Entrada

Puede tener cero o más datos de entrada — valores que recibe del mundo exterior para procesar.

📤

Salida

Debe producir al menos un resultado. Un algoritmo sin salida no resuelve ningún problema observable.

✅

Efectividad

Cada operación debe ser suficientemente básica como para ejecutarse en tiempo finito. Las operaciones deben ser realizables en la práctica, no solo en teoría.

📌 Ejemplo cotidiano

Una receta de cocina es un algoritmo: tiene entrada (ingredientes), pasos finitos (instrucciones), y produce una salida (el plato). Si dice "añadir sal al gusto", pierde precisión — y deja de ser un algoritmo formal.

1.4

Introducción al Entorno de Programación

📝

PSeInt

Diseña y prueba algoritmos en pseudocódigo. Permite ejecutar el algoritmo sin código C++. Es la herramienta de modelado del curso: siempre va primero.

⚙️

Compilador C++

Traduce código fuente C++ a código ejecutable. En este curso se utilizará g++, Code::Blocks o Dev-C++ según disponibilidad del laboratorio.

📌 Flujo de trabajo del curso

Papel (diseño) → PSeInt (verificación) → C++ (implementación) → compilar → ejecutar → verificar con tabla de simulación. El modelo siempre va antes que el código.

1.5

Anatomía de un Programa en C++

Todo programa en C++ comparte una estructura mínima. Comprender cada componente es esencial para no memorizar de forma ciega.

1#include <iostream>Incluye la librería de entrada/salida. Sin esto, cout y cin no existen.
2using namespace std;Permite usar cout y cin sin escribir std:: antes de cada uno.
3
4int main() {Función principal. La ejecución siempre comienza aquí.
5 cout << "Hola, mundo!";cout envía texto a pantalla. << es el operador de inserción.
6 return 0;Indica al SO que el programa terminó correctamente.
7}
⚠️ Error frecuente

Olvidar el punto y coma ; al final de una instrucción es el error más común en C++. Cada instrucción termina con ;, excepto las que abren bloque con {}.

1.6

Flujo Secuencial de Ejecución

Las instrucciones se ejecutan una después de otra, de arriba hacia abajo. No hay saltos, no hay decisiones, no hay repeticiones — eso viene después. Por ahora, la ejecución es estrictamente lineal.

C++
promedio.cpp 
#include <iostream>
using namespace std;

int main() {
    // 1. Declaración de variables
    double num1, num2, promedio;

    // 2. Entrada de datos
    cout << "Ingrese el primer número: ";
    cin  >> num1;
    cout << "Ingrese el segundo número: ";
    cin  >> num2;

    // 3. Proceso
    promedio = (num1 + num2) / 2.0;

    // 4. Salida
    cout << "El promedio es: " << promedio << endl;

    return 0;
}

Ejercicios de Modelado Básico

✏️ Actividades de Aplicación
1
Comprensión
Análisis de algoritmo. Identifica cuál propiedad algorítmica se viola en la siguiente secuencia: "Paso 1: Pedir un número. Paso 2: Si es negativo, volver al Paso 1. Paso 3: Si sigue siendo negativo, volver al Paso 1." Justifica tu respuesta por escrito.
2
Comprensión
Identificación de componentes. En promedio.cpp señala: (a) librería incluida, (b) instrucción de entrada, (c) instrucción de proceso, (d) instrucción de salida.
3
Aplicación
Suma de dos números. ① Pseudocódigo en PSeInt ② Diagrama de flujo ③ Tabla de simulación con valores 15 y 27 ④ Implementación en C++.
4
Aplicación
Área de un rectángulo. Programa que solicite base y altura y calcule el área (Área = base × altura). Incluye pseudocódigo y tabla de simulación con base=5.5, altura=3.2.
5
Integrador + IA
Intercambio de valores. Implementa el intercambio de dos variables con variable auxiliar. Luego busca en IA una solución sin variable auxiliar y analiza: ¿cuál es más legible? ¿cuál es más eficiente en memoria? ¿cuál preferirías y por qué? Documenta el análisis en tu bitácora.
Actividad Presencial Evaluativa — Capítulo 1
Sesión 1 · Tiempo: 45 min · Sin acceso a IA
Calcula la nota final de un estudiante: 40% exámenes, 35% prácticas, 25% tareas. Diseña el algoritmo que solicite las tres notas, calcule la nota final y la muestre con un mensaje apropiado.

Entregables: pseudocódigo en papel + tabla de simulación + implementación en C++.
  • Pseudocódigo correcto con Algoritmo/FinAlgoritmo y tipos declarados.
  • La fórmula refleja correctamente los porcentajes (40% + 35% + 25% = 100%).
  • Tabla de simulación completamente llenada con valores de prueba.
  • El programa C++ compila sin errores y produce la salida esperada.
  • El estudiante puede explicar cualquier línea al ser consultado por el docente.
