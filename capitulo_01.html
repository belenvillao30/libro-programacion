1.1

Concepto Formal de Algoritmo

Antes de escribir una sola línea de código es necesario comprender qué es lo que el código ejecuta. Un programa es la traducción de un algoritmo a un lenguaje que la máquina puede interpretar.

Definición formal

Un algoritmo es una secuencia finita, ordenada y no ambigua de pasos bien definidos que describe cómo resolver un problema. Los algoritmos son independientes del lenguaje de programación.

🤖 Reflexión sobre IA

Cuando le pides a una IA que resuelva un problema, ella genera código, no un algoritmo razonado. Si no comprendes el algoritmo, no puedes verificar si la solución es correcta o eficiente.

1.2

Características y Propiedades

🔢

Finitud

Debe terminar después de un número finito de pasos.

📌

Precisión

Cada paso debe estar definido sin ambigüedades.

📥

Entrada

Puede tener cero o más datos de entrada.

📤

Salida

Debe producir al menos un resultado observable.

✅

Efectividad

Cada operación debe ser ejecutable en tiempo finito y realizables en la práctica.

1.4

Entorno de Programación

📝

PSeInt

Diseña y ejecuta algoritmos en pseudocódigo sin necesidad de C++. Herramienta de modelado del curso.

⚙️

Compilador C++

Traduce código C++ a ejecutable. Se usará g++, Code::Blocks o Dev-C++.

📌 Flujo del curso

PSeInt → pseudocódigo correcto → C++ → compilar → ejecutar → verificar

1.5

Anatomía de un Programa en C++

1#include <iostream>Librería de entrada/salida. Sin esto cout y cin no existen.
2using namespace std;Permite usar cout y cin sin escribir std:: antes.
3
4int main() {Función principal. La ejecución siempre comienza aquí.
5cout << "Hola, mundo!";cout envía texto a pantalla. << es el operador de inserción.
6return 0;Indica al SO que el programa terminó correctamente.
7}
⚠️ Error frecuente

Olvidar el punto y coma ; al final de una instrucción es el error sintáctico más común.

1.6

Flujo Secuencial de Ejecución

C++
promedio.cpp 
#include <iostream>
using namespace std;

int main() {
    // 1. Declaración
    double num1, num2, promedio;

    // 2. Entrada
    cout << "Ingrese el primer número: ";
    cin  >> num1;
    cout << "Ingrese el segundo número: ";
    cin  >> num2;

    // 3. Proceso
    promedio = (num1 + num2) / 2.0;

    // 4. Salida
    cout << "El promedio es: " << promedio << endl;
    return 0;
}

1.7

Ejercicios de Modelado

✏️ Actividades de Aplicación
1Comprensión

Análisis de algoritmo. Identifica qué propiedad se viola: "Paso 1: Pedir número. Paso 2: Si es negativo, volver al Paso 1. Paso 3: Si sigue negativo, volver al Paso 1." Justifica.

2Comprensión

Identificación de componentes. En promedio.cpp señala: (a) librería, (b) instrucción de entrada, (c) proceso, (d) salida.

3Aplicación

Suma de dos números. ① Pseudocódigo ② Diagrama de flujo ③ Tabla de simulación con 15 y 27 ④ C++.

4Aplicación

Área de rectángulo. Programa con base y altura (decimales). Incluye pseudocódigo y tabla con base=5.5, altura=3.2.

5Integrador + IA

Intercambio de valores. Implementa con variable auxiliar. Luego busca en IA la versión sin auxiliar y analiza: ¿cuál es más legible? ¿más eficiente? ¿cuál preferirías?
