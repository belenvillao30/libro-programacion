6.1

El Ciclo while — Condición al Inicio

El while evalúa su condición antes de cada iteración. Si la condición es falsa desde el principio, el bloque nunca se ejecuta. Es el ciclo ideal cuando no sabemos de antemano cuántas repeticiones serán necesarias.

C++ · while con valor centinela
suma_centinela.cpp
#include <iostream>
#include <iomanip>
using namespace std;
int main() {
    double numero, suma = 0;
    int    cantidad = 0;

    cout << "Ingrese números (0 para terminar):\n";
    cin  >> numero;              // primera lectura ANTES del while

    while (numero != 0) {
        suma += numero;
        cantidad++;
        cin >> numero;           // leer de nuevo al final del cuerpo
    }

    if (cantidad > 0)
        cout << fixed << setprecision(2)
             << "Promedio: " << suma/cantidad << endl;
    else
        cout << "No ingresaste números.\n";
    return 0;
}
📌 Patrón leer → verificar → procesar → leer de nuevo

La primera lectura ocurre antes del ciclo para que la condición tenga un valor que evaluar. La segunda lectura está al final del cuerpo. Si olvidas esa segunda lectura tendrás un bucle infinito.

6.2

El Ciclo do-while — Condición al Final

El do-while ejecuta el bloque primero y evalúa la condición después. Garantiza al menos una ejecución aunque la condición sea falsa desde el inicio. Es la estructura natural para menús y validación de entrada.

C++ · do-while · menú con switch
menu_figuras.cpp
int opcion;
do {
    cout << "\n=== FIGURAS GEOMÉTRICAS ===\n"
         << "1. Área del círculo\n"
         << "2. Área del rectángulo\n"
         << "3. Salir\n"
         << "Opción: ";
    cin >> opcion;

    switch(opcion) {
        case 1: { double r;
                  cout<<"Radio: "; cin>>r;
                  cout<<"Área = "<<3.14159*r*r<<endl; break; }
        case 2: { double b,h;
                  cout<<"Base y altura: "; cin>>b>>h;
                  cout<<"Área = "<<b*h<<endl; break; }
        case 3: cout<<"Hasta luego.\n"; break;
        default: cout<<"Opción inválida.\n";
    }
} while (opcion != 3);
📌 do-while es el ciclo natural del menú

Un menú siempre debe mostrarse al menos una vez antes de preguntar si el usuario quiere continuar. El do-while modela exactamente eso: mostrar → ejecutar → ¿repetir? Nota el punto y coma obligatorio después del while(condición);

6.3

Comparativa: for vs while vs do-while

Ciclo¿Cuándo usarlo?CondiciónMín. ejecuciones
forNúmero de iteraciones conocido de antemano (recorrer N elementos)Al inicio0
whileNúmero desconocido, leer hasta valor centinela, condición complejaAl inicio0
do-whileMenús, validación: el bloque debe ejecutarse al menos una vezAl final1 siempre
while — puede NO ejecutarse
int x = 10;
while (x < 5) {
    cout << x; // NUNCA ejecuta
    x++;
}
// bloque ignorado
do-while — ejecuta al menos 1 vez
int x = 10;
do {
    cout << x; // imprime 10
    x++;
} while (x < 5);
// ejecutó aunque condición era falsa
🤖 IA y los bucles infinitos

La IA suele generar while(true) con un break interno. Funciona, pero oculta la condición de salida y dificulta el razonamiento y la depuración. Prefiere siempre la condición explícita en el while: comunica la intención del algoritmo y hace el código más legible.

6.4

Validación de Entrada con do-while

El patrón más práctico del do-while: pedir un dato hasta que cumpla las restricciones. Garantiza que el programa nunca trabaje con datos fuera de rango.

C++ · validación de nota
validar_nota.cpp
double nota;
do {
    cout << "Nota (0.0 – 10.0): ";
    cin  >> nota;
    if (nota < 0.0 || nota > 10.0)
        cout << "  ✗ Fuera de rango. Intente de nuevo.\n";
} while (nota < 0.0 || nota > 10.0);

// Aquí nota está garantizada en [0.0, 10.0]
cout << "Nota válida: " << nota << endl;
⚠️ Bucle infinito — el error más frecuente

Causas más comunes: olvidar actualizar la variable de control, escribir = en lugar de ==, o una condición que nunca puede volverse falsa. Siempre traza el ciclo en papel antes de ejecutarlo.

6.5

break y continue

SentenciaEfecto inmediatoCuándo usarla
breakSale del ciclo completoEncontrar el primer elemento, condición de error crítico
continueSalta el resto de la iteración actual; va a la siguienteIgnorar valores inválidos o especiales sin salir del ciclo
C++ · break y continue
break_continue.cpp
// break: encontrar primer número negativo entre 10 datos
int n; bool encontrado = false;
for (int i = 0; i < 10; i++) {
    cin >> n;
    if (n < 0) { encontrado = true; break; }
}
cout << (encontrado ? "Negativo encontrado" : "Ninguno negativo") << endl;

// continue: sumar solo positivos, ignorar 0 y negativos
double suma = 0;
for (int i = 0; i < 10; i++) {
    cin >> n;
    if (n <= 0) continue;  // salta esta iteración
    suma += n;
}

Ejercicios de Aplicación

✏️ Actividades de Aplicación
1
Comprensión
Trazar sin computadora. ¿Cuántas veces ejecuta el bloque? (a) int x=1; while(x<=16) x*=2; (b) int x=20; do{x-=3;}while(x>0); (c) while con entradas 2, 7, 5 y condición x!=5.
2
Aplicación
Validador de edad. Usa do-while para pedir la edad hasta que esté en [0, 120]. Luego clasifica: menor (0-17), adulto (18-64), adulto mayor (65+). Muestra cuántos intentos tomó.
3
Aplicación
Acumulador con centinela. Con while, leer montos de venta hasta ingresar -1. Calcular total, promedio, mayor y menor venta. Usar continue para ignorar valores de 0 en el cálculo del promedio.
4
Aplicación
Cajero automático. Saldo inicial $500. Con do-while, permitir retiros hasta que el usuario elija salir o el saldo sea 0. Validar con do-while interno que el monto sea positivo y no supere el saldo disponible.
5
Integrador + IA
Auditar un while(true). Pide a la IA un menú con while(true) y break. Reescríbelo usando do-while con condición explícita. Compara legibilidad y comportamiento. Documenta tu análisis de 5 líneas.
