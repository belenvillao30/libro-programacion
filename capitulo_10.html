10.1

La Pila de Ejecución — Cómo el Computador Gestiona las Llamadas

Cada vez que una función es llamada, el computador reserva un bloque de memoria denominado marco de pila (stack frame) que almacena las variables locales, los parámetros y la dirección de retorno. Cuando la función termina, el marco se destruye y la ejecución regresa exactamente al punto en que fue llamada.

Pila de ejecución (call stack)

Estructura LIFO (Last In, First Out) donde cada llamada a función apila un nuevo marco. La función activa siempre está en el tope de la pila. Al retornar, su marco se extrae y el control vuelve al marco anterior.

Visualización de la pila con tres llamadas encadenadas

A llama a B(2)main ← A(3) ← B(2)n=2 en B
B llama a C(1)main ← A(3) ← B(2) ← C(1)n=1 en C

📌 La pila tiene límite

Si una función se llama a sí misma sin un caso base que la detenga, la pila crece indefinidamente hasta llenarse. El programa termina con un error llamado desbordamiento de pila (stack overflow). Por eso toda función recursiva necesita al menos un caso base.

10.2

¿Qué es la Recursión?

Una función es recursiva cuando su definición incluye una llamada a sí misma con parámetros que se acercan progresivamente a la condición de parada. La recursión no es magia: es la pila de ejecución trabajando de forma anidada.

Dos componentes obligatorios de toda función recursiva

Caso base: condición en la que la función no se llama a sí misma y retorna un valor conocido. Sin él la recursión es infinita. — Caso recursivo: la función se llama con un argumento más cercano al caso base, dividiendo el problema en una versión más pequeña del mismo problema.

C++ · estructura general de función recursiva
estructura_recursiva.cpp 
tipo_retorno funcionRecursiva(parametros) {
    // ① CASO BASE — detiene la recursión
    if (condicion_base) {
        return valor_conocido;
    }

    // ② CASO RECURSIVO — llama con parámetro más cercano al caso base
    return funcionRecursiva(parametros_reducidos);
}
⚠️ Errores comunes en recursión

Sin caso base: la función se llama infinitamente → stack overflow. Caso base inalcanzable: los parámetros nunca llegan a la condición base. Parámetro que no avanza: se llama siempre con el mismo valor → recursión infinita. Antes de codificar, verifica que el caso recursivo siempre acerca el parámetro al caso base.

10.3

Ejemplo Clásico: Factorial

El factorial de un número entero no negativo n se define matemáticamente como: n! = n × (n−1) × (n−2) × … × 1, con el caso base 0! = 1. Esta definición es naturalmente recursiva: n! = n × (n−1)!

Versión recursiva 
long long factorialR(int n) {
  if (n == 0) return 1;  // caso base
  return n * factorialR(n - 1);
}
Versión iterativa equivalente 
long long factorialI(int n) {
  long long res = 1;
  for (int i = 2; i <= n; i++)
    res *= i;
  return res;
}

Traza de pila — factorial(4)

LlamadaEstado de la pilaRetorna

Desapilando...1 × 1 = 1 · 2×1=2 · 3×2=64 × 6 = 24
10.4

Fibonacci — Recursión con Dos Llamadas

La secuencia de Fibonacci: 0, 1, 1, 2, 3, 5, 8, 13, 21... se define como F(n) = F(n−1) + F(n−2) con casos base F(0) = 0 y F(1) = 1. Es el ejemplo de recursión múltiple: cada llamada genera dos llamadas hijas, formando un árbol de llamadas.

C++ · Fibonacci recursivo con validación de entrada
fibonacci.cpp 
#include <iostream>
using namespace std;

long long fibonacci(int n) {
    if (n == 0) return 0;   // caso base 1
    if (n == 1) return 1;   // caso base 2
    return fibonacci(n - 1) + fibonacci(n - 2);  // dos llamadas recursivas
}

int main() {
    int n;
    do {
        cout << "Ingresa n (0-40): ";
        cin  >> n;
    } while (n < 0 || n > 40);  // límite: Fibonacci recursivo es lento para n>40

    cout << "F(" << n << ") = " << fibonacci(n) << endl;

    // Imprimir toda la secuencia hasta n
    cout << "Secuencia: ";
    for (int i = 0; i <= n; i++)
        cout << fibonacci(i) << (i < n ? " " : "\n");
    return 0;
}
⚠️ Costo exponencial de Fibonacci recursivo

Fibonacci recursivo tiene costo O(2ⁿ): fibonacci(5) se calcula 1 vez, pero fibonacci(2) se recalcula 3 veces. Para n=40 hay ~330 millones de llamadas. Por eso se valida que n≤40. En la práctica, Fibonacci iterativo o con memoización es preferible; aquí se usa para ilustrar el árbol de llamadas.

10.5

Suma de Dígitos — Recursión sobre Datos

No toda recursión opera sobre contadores. La suma de dígitos muestra cómo dividir un dato (el número) en partes más pequeñas hasta alcanzar el caso base.

C++ · suma de dígitos recursiva
suma_digitos.cpp 
// sumaDigitos(1234) = 4 + sumaDigitos(123)
//                   = 4 + 3 + sumaDigitos(12)
//                   = 4 + 3 + 2 + sumaDigitos(1)
//                   = 4 + 3 + 2 + 1 = 10

int sumaDigitos(int n) {
    if (n < 10) return n;           // caso base: un solo dígito
    return (n % 10) + sumaDigitos(n / 10);   // último dígito + resto
}

int main() {
    int n;
    cout << "Número: "; cin >> n;
    cout << "Suma de dígitos: " << sumaDigitos(n) << endl;
    return 0;
}

Llamadan % 10n / 10Retorna

sumaDigitos(25)525 + sumaDigitos(2)
sumaDigitos(2) ← caso base——2
Desapilando: 5+2=7 · 3+7=1010
10.6

Recursión vs. Iteración — ¿Cuándo usar cada una?

AspectoRecursiónIteración
LegibilidadElegante para problemas naturalmente divisibles (factorial, árboles)Más directa y fácil de seguir línea a línea
MemoriaUsa la pila — riesgo de stack overflow para n grandesEficiente: solo variables locales del bucle
VelocidadPuede ser lenta si hay llamadas repetidas (Fibonacci naïve)Generalmente más rápida para problemas lineales
DepuraciónMás difícil: múltiples marcos activos simultáneamenteMás sencilla: flujo lineal, fácil de rastrear
Cuándo es naturalÁrboles, grafos, divide y vencerás, algoritmos matemáticosSumas, búsquedas lineales, procesamiento de arreglos
🤖 IA y la recursión

La IA tiende a proponer soluciones recursivas para problemas donde la iteración es más eficiente. Antes de aceptar una solución recursiva de la IA, pregúntate: (1) ¿El problema se divide naturalmente en subproblemas del mismo tipo? (2) ¿El n de entrada puede ser grande (riesgo de stack overflow)? (3) ¿Hay llamadas redundantes (riesgo de costo exponencial)? Si alguna respuesta es preocupante, pide la versión iterativa y compara.

Aplicación
potencia recursiva. Implementa double potencia(double base, int exp) donde exp ≥ 0. Caso base: exp==0 → retorna 1. Caso recursivo: base × potencia(base, exp−1). Prueba con (2,10)=1024, (3,0)=1, (2.5,3)=15.625. Después escribe la versión iterativa y compara el número de operaciones.

Aplicación
esPalindromo. Implementa bool esPalindromo(const string& s, int ini, int fin). Caso base: ini≥fin → true (cadena de 0 o 1 caracteres es palíndromo). Caso recursivo: si s[ini]==s[fin] → esPalindromo(s, ini+1, fin-1), si no → false. Prueba con "radar", "hello", "anilina", "a". ¿Por qué se usa const string& y no copia?

Aplicación

Actividad Presencial Evaluativa — Capítulo 10
Sesión 10 · Tiempo: 45 min · Sin acceso a IA ni a internet
Implementa un programa con tres funciones recursivas: int mcd(int a, int b) que calcula el Máximo Común Divisor usando el algoritmo de Euclides (caso base: b==0 → retorna a; caso recursivo: mcd(b, a%b)); int sumaHasta(int n) que calcula 1+2+…+n recursivamente; y bool esPrimo(int n, int divisor) que verifica primalidad recursiva (caso base: divisor*divisor > n → true; recursivo: n%divisor==0 → false, si no → esPrimo(n, divisor+1)). El main lee n, muestra mcd(n,100), sumaHasta(n), esPrimo(n,2) y una tabla de primos hasta n.
  • Caso base y caso recursivo correctamente identificados y comentados en cada función.
  • Las tres funciones producen resultados correctos verificados con al menos 3 valores de n.
  • Tabla de traza manual para mcd(48, 18) mostrando cada llamada.
  • Comparación escrita: ¿cuál de las tres sería más eficiente en versión iterativa y por qué?
  • Validación de entrada: n debe ser entero positivo ≥ 2.
