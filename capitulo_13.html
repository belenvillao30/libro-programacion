13.1

Búsqueda Binaria — Dividir y Conquistar en O(log n)

La búsqueda binaria requiere un arreglo ordenado. En cada paso, compara el elemento buscado con el elemento central y descarta la mitad del arreglo donde el elemento no puede estar. Con 1024 elementos, encuentra cualquier valor en máximo 10 comparaciones.

Por qué funciona — reducción por mitades

Cada comparación divide el espacio de búsqueda a la mitad. Después de k comparaciones, quedan n/2ᵏ elementos. El algoritmo termina cuando 1 ≥ n/2ᵏ, es decir, k = log₂n. Por eso el costo es O(log n) — para 1 millón de elementos, máximo 20 comparaciones.

C++ · búsqueda binaria iterativa — retorna índice o -1
busqueda_binaria.cpp 
// REQUIERE: arreglo ordenado ascendentemente
int busquedaBinaria(const double arr[], int n, double obj) {
    int izq=0, der=n-1;
    while (izq <= der) {
        int mid = (izq + der) / 2;   // índice central
        if      (arr[mid] == obj) return mid;   // ¡encontrado!
        else if (arr[mid]  < obj) izq = mid + 1; // buscar en mitad derecha
        else                      der = mid - 1; // buscar en mitad izquierda
    }
    return -1;   // no encontrado
}

⚠️ Búsqueda binaria solo funciona con arreglos ordenados

Si el arreglo no está ordenado, la búsqueda binaria puede retornar −1 aunque el elemento exista. Siempre ordena primero o usa búsqueda lineal si no puedes garantizar el orden. Este es el error más común con este algoritmo.

13.2

Ordenamiento por Selección

En cada pasada, selecciona el mínimo del subarreglo no ordenado y lo coloca en su posición definitiva. A diferencia del burbuja, hace exactamente n−1 pasadas con a lo sumo 1 intercambio por pasada.

C++ · ordenamiento por selección
seleccion.cpp 
void seleccion(double arr[], int n) {
    for (int i=0; i<n-1; i++) {
        int idxMin = i;   // supone que el mínimo está en la posición i
        for (int j=i+1; j<n; j++)
            if (arr[j] < arr[idxMin]) idxMin = j;  // actualizar mínimo
        if (idxMin != i) {   // solo intercambiar si encontramos un mínimo menor
            double tmp = arr[i]; arr[i] = arr[idxMin]; arr[idxMin] = tmp;
        }
    }
}

13.3

Ordenamiento por Inserción

Funciona como ordenar cartas en la mano: toma un elemento de la parte no ordenada y lo inserta en la posición correcta de la parte ya ordenada. Muy eficiente para arreglos casi ordenados (O(n) en el mejor caso).

C++ · ordenamiento por inserción
insercion.cpp 
void insercion(double arr[], int n) {
    for (int i=1; i<n; i++) {
        double clave = arr[i];   // elemento a insertar
        int    j     = i - 1;
        // desplazar hacia la derecha los elementos mayores que la clave
        while (j >= 0 && arr[j] > clave) {
            arr[j+1] = arr[j];
            j--;
        }
        arr[j+1] = clave;   // colocar la clave en su posición
    }
}
13.4

Comparativa — Cuándo Usar Cada Algoritmo

AlgoritmoMejor casoPeor casoIntercambiosCuándo usar
Búsqueda linealO(1)O(n)—Arreglos no ordenados o pequeños
Búsqueda binariaO(1)O(log n)—Arreglos ordenados de cualquier tamaño
BurbujaO(n)*O(n²)O(n²)Solo didáctico o arreglos muy pequeños
SelecciónO(n²)O(n²)O(n)Cuando el costo de intercambio es alto
InserciónO(n)O(n²)O(n²)Arreglos casi ordenados o de llegada continua

* Burbuja con flag de optimización. El O(n) mejor caso de inserción es real y frecuente en datos reales.

🤖 IA y el ordenamiento

La IA probablemente sugerirá std::sort() de <algorithm> — una función estándar que usa Introsort (O(n log n) garantizado). Esto es correcto para proyectos reales. En este capítulo implementamos los algoritmos manualmente para entender cómo y por qué cada uno tiene el costo que tiene. Una vez dominado el concepto, usar std::sort() es la decisión correcta en producción.
