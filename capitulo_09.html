9.1

El Problema: Modificar la Variable Original

Hasta ahora todas nuestras funciones recibían copias de los datos. Pero a veces necesitamos que la función modifique directamente la variable del llamador, no una copia. El intercambio de dos variables es el ejemplo clásico: parece que funciona, pero sin referencia no cambia nada fuera de la función.

C++ · intento fallido — intercambio sin referencia
swap_falla.cpp 
void intercambiar(int a, int b) {   // a y b son COPIAS locales
    int temp = a;
    a = b;
    b = temp;
    // los cambios mueren aquí: x e y en main quedan intactos
}

int main() {
    int x = 5, y = 10;
    intercambiar(x, y);
    cout << x << " " << y;   // imprime: 5 10  ← ¡no cambió nada!
    return 0;
}
⚠️ El engaño del paso por valor

La función parece intercambiar, pero trabaja con copias locales. Al terminar la función esas copias se destruyen y x e y en main quedan intactos. La solución: pasar por referencia con el operador &, que convierte el parámetro en un alias del original.

9.2

Paso por Valor — Trabaja con una Copia

Es el comportamiento predeterminado en C++. La función recibe una copia independiente del argumento. Cualquier modificación dentro de la función no afecta a la variable original. Ideal cuando la función solo necesita leer el dato sin modificarlo.

Cuándo usar paso por valor

Cuando la función solo lee el dato y no debe modificarlo. Para tipos simples (int, double, bool, char) el costo de copiar es despreciable. El compilador puede optimizarlo automáticamente con copy elision.

C++ · paso por valor — el original queda protegido
por_valor.cpp 
double calcularIVA(double precio) {    // precio es copia independiente
    precio *= 1.12;     // modifica la copia, NO el original
    return precio;
}

int main() {
    double subtotal = 100.0;
    double total    = calcularIVA(subtotal);
    cout << "Subtotal: " << subtotal << endl;   // 100.0 — sin cambios
    cout << "Con IVA:  " << total    << endl;   // 112.0
    return 0;
}
9.3

Paso por Referencia — Trabaja con el Original

El operador & en el parámetro convierte la variable en un alias: apunta a la misma posición de memoria que la variable original. Toda modificación dentro de la función afecta directamente al argumento del llamador.

Sintaxis

tipo& nombre — el & puede ir pegado al tipo (int& x) o al nombre (int &x); ambos son válidos en C++. Por convención académica se prefiere pegado al tipo.

C++ · intercambio correcto con referencia
swap_correcto.cpp 
#include <iostream>
using namespace std;

void intercambiar(int& a, int& b) {    // & = alias del original
    int temp = a;
    a = b;
    b = temp;
    // modifica directamente x e y en main
}

int main() {
    int x = 5, y = 10;
    cout << "Antes:   x=" << x << "  y=" << y << endl;   // 5  10
    intercambiar(x, y);
    cout << "Después: x=" << x << "  y=" << y << endl;   // 10  5
    return 0;
}

📌 Alias = misma dirección de memoria

El alias a no es una nueva variable: es otro nombre para la dirección de x. Escribir en a es literalmente escribir en x. Por eso el cambio persiste al salir de la función. No se copió ningún dato.

🤖 IA y el paso por referencia

La IA frecuentemente genera funciones que reciben todo por referencia "por eficiencia". Esa práctica es incorrecta como regla general. Solo usa & cuando la función necesita modificar el argumento o devolver múltiples valores. Para datos de entrada de solo lectura, usa const& (§9.5) o paso por valor para tipos simples. Siempre revisa la elección de la IA y justifica cada parámetro.

9.5

Parámetros const& — Eficiencia sin Riesgo

Para pasar objetos grandes (como string) que la función solo necesita leer, const& combina lo mejor de ambos mundos: evita la copia (eficiencia de referencia) y garantiza que el original no puede ser modificado (seguridad de valor). El compilador genera error si intentas escribir en él.

Sin const& — ineficiente 
void mostrar(string nombre) {
  // copia todo el string
  cout << nombre;
}
Con const& — eficiente y seguro 
void mostrar(const string& nombre) {
  // alias sin copia, protegido
  cout << nombre;
}
C++ · const& en práctica — protección garantizada por el compilador
const_ref.cpp 
// Solo calcula — no necesita modificar precio
double calcularDescuento(const double& precio, double pct) {
    return precio * pct / 100.0;
    // precio = 0;  ← ERROR de compilación: violación de const
}

// Reporte de bienvenida — string grande, solo lectura
void imprimirBienvenida(const string& nombre, const string& carrera) {
    cout << "Bienvenido/a, " << nombre << "\nCarrera: " << carrera << endl;
}

9.6

Devolver Múltiples Resultados con Referencias

Una función en C++ solo puede retornar un valor con return. Cuando necesitamos varios resultados simultáneos, la solución clásica son los parámetros de salida por referencia: la función los escribe directamente en las variables del llamador.

C++ · calcularEstadisticas — tres valores de salida por referencia
estadisticas_ref.cpp 
#include <iostream>
#include <iomanip>
using namespace std;

// n: ENTRADA por valor · minV, maxV, prom: SALIDA por &
void calcularEstadisticas(int n, double& minV, double& maxV, double& prom);

int main() {
    int    n = 5;
    double minimo, maximo, promedio;   // sin inicializar — la función las llena
    calcularEstadisticas(n, minimo, maximo, promedio);
    cout << fixed << setprecision(2)
         << "Mín: " << minimo << "  Máx: " << maximo
         << "  Prom: " << promedio << endl;
    return 0;
}

void calcularEstadisticas(int n, double& minV, double& maxV, double& prom) {
    double suma = 0, dato;
    cout << "Ingresa " << n << " números:\n";
    cin  >> dato;
    minV = maxV = dato;    // primer dato inicializa min y max
    suma = dato;
    for (int i = 1; i < n; i++) {
        cin >> dato;
        if (dato < minV) minV = dato;
        if (dato > maxV) maxV = dato;
        suma += dato;
    }
    prom = suma / n;       // escribe en la variable de main a través del alias
}
📌 Convención: documentar roles de parámetros

En funciones con parámetros mixtos, añadir /*in*/ o /*out*/ junto al prototipo hace el contrato de la función auto-documentado. El lector sabe de un vistazo qué parámetros son datos de entrada y cuáles son resultados de salida, sin tener que leer el cuerpo de la función.
