Práctica 2 — Tipos, Variables y Expresiones en Contexto Real

Elegir el tipo correcto · Calcular con precisión · Formatear la salida · Detectar errores de tipo

Duración3 horas (3 ACD)
ModalidadIndividual + parejas
EntregaPapel + digital (.cpp)
Capítulo baseCap. 2
F1 Análisis de tipos (0–20 min) F2 PSeInt (20–50 min) F3 C++ con formato (50–90 min) F4 Cazador de errores (90–130 min) F5 Análisis con IA (130–160 min) F6 Cierre (160–180 min)
🎯 Problema central de la sesión
Una farmacia necesita un sistema de facturación básico. El programa debe registrar: nombre del cliente (cadena), código del producto (entero), precio unitario, cantidad comprada, porcentaje de descuento por volumen (si compra 5 o más unidades, aplica 10%). Calcular y mostrar: subtotal, descuento aplicado, IVA (12%) y total a pagar — todo con 2 decimales y etiquetas alineadas.

IVA constante: const double IVA = 0.12;

F3
Implementación en C++ con salida formateada — Individual
⏱ 50 – 90 minutos

Traduce el pseudocódigo a C++. Presta especial atención a: tipos de datos, la constante IVA, y el formato de salida con fixed + setprecision(2).

C++ · Referencia
factura_farmacia.cpp 
#include <iostream>
#include <iomanip>
#include <string>
using namespace std;

int main() {
    // --- Constante institucional ---
    const double IVA = 0.12;

    // --- Variables de entrada ---
    string nombre_cliente;
    int    codigo_producto, cantidad;
    double precio_unitario;

    // --- Variables de cálculo ---
    double subtotal, descuento, iva_monto, total;

    // --- Entrada de datos ---
    cout << "=== FARMACIA ESPAM MFL ===\n";
    cout << "Nombre del cliente:  ";
    getline(cin, nombre_cliente);   // getline para leer espacios
    cout << "Código del producto: ";
    cin  >> codigo_producto;
    cout << "Precio unitario:     $";
    cin  >> precio_unitario;
    cout << "Cantidad:            ";
    cin  >> cantidad;

    // --- Proceso ---
    subtotal = precio_unitario * cantidad;

    // Descuento por volumen (5+ unidades → 10%)
    if (cantidad >= 5)
        descuento = subtotal * 0.10;
    else
        descuento = 0.0;

    iva_monto = (subtotal - descuento) * IVA;
    total     = subtotal - descuento + iva_monto;

    // --- Salida formateada ---
    cout << fixed << setprecision(2);
    cout << "\n=== FACTURA ===\n";
    cout << "Cliente:    " << nombre_cliente     << endl;
    cout << "Código:     " << codigo_producto     << endl;
    cout << "Subtotal:   $" << setw(8) << subtotal  << endl;
    cout << "Descuento:  $" << setw(8) << descuento << endl;
    cout << "IVA (12%):  $" << setw(8) << iva_monto << endl;
    cout << "TOTAL:      $" << setw(8) << total     << endl;

    return 0;
}
📌 getline vs cin

cin >> nombre se detiene en el primer espacio. getline(cin, nombre) lee toda la línea. Para nombres con apellido, siempre usa getline. Si mezclas cin >> y getline, agrega cin.ignore(); entre ellos.

F4
Cazador de errores de tipo — Parejas
⏱ 90 – 130 minutos

El siguiente código tiene 5 errores relacionados con tipos de datos. Identifícalos, explica por qué son errores y propón la corrección. No lo ejecutes todavía — analiza el código visualmente primero.

C++ · CON ERRORES — encontrar y corregir
errores.cpp 
#include <iostream>
using namespace std;

int main() {
    int precio = 15.99;           // Error 1: ?
    int cantidad = 3;
    int total = precio * cantidad;  // Error 2: ?

    int a = 7, b = 2;
    double resultado = a / b;       // Error 3: ?

    char nombre = "Juan";          // Error 4: ?

    double pct = 12 / 100;         // Error 5: ?

    cout << total << endl;
    cout << resultado << endl;
    cout << pct << endl;
    return 0;
}
A
Analiza cada error
Para cada línea marcada con "Error N": (1) Describe qué está mal, (2) Indica qué valor incorrecto se obtendría, (3) Escribe la corrección. Documenta en papel antes de verificar en el compilador.
B
Verificación con el compilador
Compila el código original. ¿Cuáles errores detectó el compilador? ¿Cuáles pasaron sin advertencia pero producen resultados incorrectos? Esta distinción es fundamental.
⚠ Reflexión clave

Algunos errores de tipo no generan error de compilación — el programa "funciona" pero produce valores equivocados silenciosamente. Estos son los más peligrosos porque no hay mensaje que te avise. Solo la comprensión del tipo te permite detectarlos.
