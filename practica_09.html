🔬 Unidad III · Sesión Práctica 9

Sistema de Conversión de Unidades Modular

paso por valor · paso por referencia · const& · múltiples salidas · validador reutilizable

Duración180 min (3h)
ModalidadIndividual → Parejas
EntregableCódigo .cpp + bitácora
F1 · Diseño F2 · Pseudocódigo F3 · C++ Individual F4 · Extensión F5 · Análisis IA F6 · Bitácora
Objetivo de la Sesión
El estudiante construye un sistema de conversión de unidades completamente modular donde cada grupo de conversiones devuelve múltiples resultados usando parámetros por referencia, las funciones de solo lectura usan const&, y la validación de entrada es una función reutilizable en todos los contextos del menú. main() solo orquesta: no calcula nada directamente.
🔬 Contexto del Problema
El laboratorio de física de ESPAM MFL necesita un conversor de unidades para sus prácticas. Menú con do-while + switch: 1) Temperatura (°C → °F, K) 2) Longitud (m → km, cm, mm, pulgadas, pies) 3) Masa (kg → g, lb, oz) 4) Salir. Cada conversión es una función void con las salidas por referencia. La función leerPositivo(const string&) valida todas las entradas numéricas del programa.
F1
Diseño de Firmas — Decidir Valor vs. Referencia en Papel
0 – 20 min · SIN computadora

Antes de abrir el IDE, decide el tipo de paso de cada parámetro de cada función. Este es el trabajo de diseño más importante de la práctica.

1
Tabla de decisión de parámetros
Para cada función, llena una tabla con tres columnas: nombre del parámetro · tipo de paso (valor / referencia / const&) · justificación. Funciones a analizar: convertirTemperatura, convertirLongitud, convertirMasa, leerPositivo, mostrarMenu. Recuerda la regla: si la función necesita escribir en el parámetro → referencia; si solo lee → valor o const&; si lee un objeto grande → const&.
2
Fórmulas de conversión en papel
Escribe y verifica todas las fórmulas: °F = (°C × 9/5) + 32 · K = °C + 273.15 · km = m÷1000 · cm = m×100 · mm = m×1000 · in = m×39.3701 · ft = m×3.28084 · g = kg×1000 · lb = kg×2.20462 · oz = kg×35.274. Comprueba con valores conocidos: 0°C = 32°F = 273.15 K · 1 m = 100 cm · 1 kg = 1000 g.
3
Casos de prueba manuales
Calcula a mano y anota los resultados esperados: 100°C → ? °F, ? K · 1.5 m → km, cm, mm, in, ft · 2.0 kg → g, lb, oz. Estos valores son tu contrato de verificación en F3: si el programa arroja otro resultado, hay un error en el código.

F3
Implementación Completa en C++ — Individual
55 – 105 min
C++ · Implementación completa del conversor de unidades
conversor_unidades.cpp 
#include <iostream>
#include <iomanip>
#include <string>
using namespace std;

// ─── PROTOTIPOS ─────────────────────────────────────────────────
void   mostrarMenu();
double leerPositivo(const string& msg);
void   convertirTemperatura(double c, double& f, double& k);
void   convertirLongitud(double m, double& km, double& cm,
                         double& mm, double& in, double& ft);
void   convertirMasa(double kg, double& g, double& lb, double& oz);

// ─── MAIN ────────────────────────────────────────────────────────
int main() {
    int opcion;
    do {
        mostrarMenu();
        cin >> opcion;
        switch (opcion) {
            case 1: {
                double f, k;
                double c = leerPositivo("  Temperatura en °C: ");
                convertirTemperatura(c, f, k);
                cout << fixed << setprecision(2)
                     << "  → " << f << "°F   " << k << " K
";
                break; }
            case 2: {
                double km, cm, mm, pin, ft;
                double m = leerPositivo("  Distancia en metros: ");
                convertirLongitud(m, km, cm, mm, pin, ft);
                cout << fixed << setprecision(4)
                     << "  → " << km << " km | " << cm << " cm | "
                     << mm << " mm | " << pin << " in | " << ft << " ft
";
                break; }
            case 3: {
                double g, lb, oz;
                double kg = leerPositivo("  Masa en kg: ");
                convertirMasa(kg, g, lb, oz);
                cout << fixed << setprecision(3)
                     << "  → " << g << " g | " << lb << " lb | " << oz << " oz
";
                break; }
            case 4:
                cout << "Hasta luego.
"; break;
            default:
                cout << "Opción inválida. Intente de nuevo.
";
        }
    } while (opcion != 4);
    return 0;
}

// ─── DEFINICIONES ────────────────────────────────────────────────
void mostrarMenu() {
    cout << "
╔══ CONVERSOR ESPAM MFL ══╗
"
         << "║ 1. Temperatura (°C)     ║
"
         << "║ 2. Longitud (m)         ║
"
         << "║ 3. Masa (kg)            ║
"
         << "║ 4. Salir               ║
"
         << "╚═══════════════════════╝
"
         << "Opción: ";
}

double leerPositivo(const string& msg) {
    double v;
    do {
        cout << msg;
        cin  >> v;
        if (v <= 0) cout << "  ✗ Debe ser un valor positivo.
";
    } while (v <= 0);
    return v;
}

void convertirTemperatura(double c, double& f, double& k) {
    f = (c * 9.0 / 5.0) + 32.0;
    k = c + 273.15;
}

void convertirLongitud(double m, double& km, double& cm,
                       double& mm, double& in, double& ft) {
    km = m / 1000.0;
    cm = m * 100.0;
    mm = m * 1000.0;
    in = m * 39.3701;
    ft = m * 3.28084;
}

void convertirMasa(double kg, double& g, double& lb, double& oz) {
    g  = kg * 1000.0;
    lb = kg * 2.20462;
    oz = kg * 35.274;
}
📋 Lista de verificación antes de avanzar a F4

☐ Cada parámetro de salida usa &. ☐ Los parámetros de entrada de tipos simples van por valor. ☐ El mensaje en leerPositivo usa const string&. ☐ main() no calcula nada directamente. ☐ Los tres casos del F1 producen los resultados esperados.

Extensión en Parejas
105 – 140 min

Intercambia tu archivo con un compañero. Lee su código durante 5 minutos y luego implementen juntos ambas extensiones sobre la versión que consideren mejor estructurada.

A
Conversión inversa de temperatura
Agrega opción 5: convertir de °F a °C y K. Escribe void convertirTempInversa(double f, double& c, double& k). Fórmulas: C = (F − 32) × 5/9 · K = C + 273.15. Observa que el parámetro que antes era salida (celsius) ahora es entrada, y el que era entrada (fahrenheit) ahora es salida. Documenta este "giro de roles" en tu bitácora con al menos 3 líneas de análisis.
B
Historial de conversiones
Crea void actualizarHistorial(int& totalConv, double& sumaEntradas, double entrada) que acumula cuántas conversiones se han realizado y la suma total de los valores de entrada. Llámala desde cada case del menú. Al ejecutar la opción Salir, llama a void mostrarHistorial(int total, double suma) que imprime el resumen. ¿Por qué total y sumaEntradas usan & y entrada no?
F5
Análisis Crítico con IA — Documentado
140 – 163 min
🤖 Protocolo de Análisis IA · Sesión 9

Muestra a la IA tu función convertirLongitud y pregúntale: "¿Cómo harías esta misma función sin usar parámetros por referencia?" Probablemente sugiera un struct o retornar varios valores con std::tuple. Responde por escrito estas 4 preguntas: (1) ¿La alternativa es más o menos legible para un estudiante de primer año? (2) ¿Requiere conocimiento nuevo que aún no hemos visto? (3) ¿En qué tipo de proyecto preferirías la versión con referencias y en cuál la alternativa? (4) ¿Hay alguna situación donde sea obligatorio usar referencia en lugar de retorno? Firma tu análisis con nombre y fecha.

📌 Límite de uso de IA en esta práctica

Puedes usar la IA para entender errores del compilador, verificar fórmulas de conversión o explorar alternativas de diseño (F5). No puedes pedirle que escriba las funciones de conversión de tu entrega. Todo el código entregado debe ser tuyo y debes poder explicar cada línea en una revisión oral.
