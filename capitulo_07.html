Resultado de Aprendizaje — Capítulo 7
El estudiante combina estructuras de control (selectivas y repetitivas) para resolver problemas algorítmicos de complejidad real, aplicando una estrategia de diseño antes de codificar y eligiendo la estructura adecuada para cada parte del problema.
7.1

Estrategia de Diseño antes de Codificar

Cuando un problema requiere múltiples estructuras anidadas, escribir código directamente produce errores difíciles de localizar. El proceso correcto tiene cuatro pasos:

PasoAcciónHerramienta
1. LeerIdentificar entradas, salidas y restricciones del problemaLápiz y papel
2. DescomponerDividir el problema en subproblemas más pequeñosLista de tareas
3. ModelarEscribir pseudocódigo o diagrama de flujoPSeInt / papel
4. CodificarTraducir el modelo a C++ verificadoIDE
📌 La regla del 70/30

En problemas complejos, dedica el 70% del tiempo a diseñar (pasos 1–3) y solo el 30% a codificar (paso 4). El código que surge de un buen modelo tarda menos en escribirse y tiene menos errores.

7.2

Ciclos Anidados

Un ciclo dentro de otro multiplica las iteraciones. El ciclo externo controla las "filas" y el interno las "columnas". Por cada iteración del externo, el interno completa todas las suyas.

C++ · ciclos anidados · tabla de multiplicar
tabla_mult.cpp
#include <iostream>
#include <iomanip>
using namespace std;
int main() {
    for (int i = 1; i <= 5; i++) {       // ciclo externo: filas 1..5
        for (int j = 1; j <= 5; j++) {   // ciclo interno: columnas 1..5
            cout << setw(4) << i * j;
        }
        cout << endl;                    // salto de línea al terminar fila
    }
    return 0;
}

⚠️ Ciclos anidados e IA

La IA genera ciclos anidados correctamente para casos simples, pero en problemas con condiciones de salida complejas puede mezclar variables de control o usar el mismo nombre de variable (i, i) produciendo comportamiento indefinido. Siempre usa nombres descriptivos distintos.

7.3

if dentro de Ciclos — Procesamiento Condicional

Combinar if-else con ciclos es el patrón más frecuente en programación real. Permite clasificar, filtrar y acumular selectivamente.

C++ · clasificador de calificaciones
clasificar_notas.cpp
int aprobados = 0, reprobados = 0;
double nota, sumaAprobados = 0;

for (int i = 1; i <= 30; i++) {
    cout << "Nota estudiante " << i << ": ";
    cin  >> nota;

    if (nota >= 5.0) {
        aprobados++;
        sumaAprobados += nota;
    } else {
        reprobados++;
    }
}

cout << "Aprobados: " << aprobados << endl;
cout << "Reprobados: " << reprobados << endl;
if (aprobados > 0)
    cout << "Prom. aprobados: " << sumaAprobados/aprobados << endl;
7.4

switch + do-while — El Menú Completo

La combinación do-while + switch es el patrón estándar para cualquier sistema con menú. El do-while garantiza que el menú aparezca siempre, y el switch despacha la opción elegida.

C++ · sistema de empleados · menú completo
sistema_empleados.cpp
#include <iostream>
#include <iomanip>
using namespace std;
int main() {
    int    opcion, cantidad = 0;
    double salario, totalSalarios = 0;
    double maxSalario = 0;

    do {
        cout << "\n=== NÓMINA ESPAM MFL ===\n"
             << "1. Registrar empleados\n"
             << "2. Ver resumen\n"
             << "3. Salir\n  Opción: ";
        cin >> opcion;

        switch(opcion) {
            case 1: {
                int n;
                do { cout<<"¿Cuántos empleados? "; cin>>n; }
                while (n <= 0);

                for (int i = 1; i <= n; i++) {
                    do { cout<<"Salario "<<i<<": "; cin>>salario; }
                    while (salario <= 0);

                    cantidad++;
                    totalSalarios += salario;
                    if (salario > maxSalario) maxSalario = salario;
                }
                break;
            }
            case 2:
                if (cantidad == 0) { cout<<"Sin registros.\n"; break; }
                cout << fixed << setprecision(2)
                     << "Empleados: " << cantidad << endl
                     << "Promedio:  $" << totalSalarios/cantidad << endl
                     << "Máx:       $" << maxSalario << endl;
                break;
            case 3: cout<<"Hasta luego.\n"; break;
            default: cout<<"Opción inválida.\n";
        }
    } while (opcion != 3);
    return 0;
}
🤖 Estructuras anidadas e IA — riesgo de complejidad excesiva

Ante problemas con múltiples condiciones anidadas, la IA suele generar código funcionalmente correcto pero con anidamiento innecesariamente profundo o condiciones redundantes. Siempre pregúntate: ¿se puede simplificar con un break anticipado o reestructurando la condición?

7.5

Guía de Elección de Estructura

Ante un problema nuevo, hazte estas preguntas en orden para elegir las estructuras correctas:

PreguntaRespuesta → Estructura
¿El bloque se repite?Sí → ciclo. No → secuencia o condicional.
¿Cuántas veces se repite?Número fijo conocido → for. Desconocido → while o do-while.
¿Debe ejecutarse al menos una vez?Sí → do-while. No garantizado → while.
¿El bloque depende de una condición?Binaria → if-else. Múltiples valores enteros → switch.
¿Se necesita salir anticipadamente?Usa break con criterio. Evita múltiples puntos de salida.
7.6

Ejercicios de Integración

✏️ Actividades de Integración
1
Comprensión
Analizar código. Dado el sistema de empleados del §7.4, traza manualmente con: 2 empleados (salarios $800 y $1200), luego elegir "Ver resumen". ¿Cuántas veces entra cada ciclo? ¿Cuál es el valor de todas las variables al final?
2
Aplicación
Sistema de calificaciones por materia. Menú con do-while+switch: ingresar notas de N estudiantes (N validado con do-while), calcular promedio, contar aprobados/reprobados, mostrar reporte. Usar for para recorrer N estudiantes y if-else para clasificar.
3
Aplicación
Pirámide de asteriscos. Con ciclos anidados, imprimir una pirámide de N filas (N validado con do-while). La fila i contiene 2i-1 asteriscos centrados. Añadir opción al menú para elegir carácter de relleno.
4
Integrador + IA
Sistema de inventario básico. Menú (do-while): (1) Registrar producto (nombre tipo string, precio validado), (2) Calcular total inventario, (3) Buscar el producto más caro, (4) Salir. Diseña primero en papel, luego codifica. Pide ayuda a la IA solo para entender cómo usar string con cin.
Actividad Presencial Evaluativa — Capítulo 7
Sesión 7 · Tiempo: 50 min · Sin acceso a IA
Sistema de registro de ventas diarias: el usuario ingresa el nombre del vendedor y sus ventas del día (N ventas, N validado). Al final mostrar: total vendido por vendedor, si supera la meta de $500, ranking (el sistema maneja hasta 5 vendedores en sesiones de menú repetidas). Menú principal: registrar vendedor, ver ranking, salir.

  • Menú con do-while + switch.
  • Validación con do-while para N ventas y cada monto.
  • Ciclo for correcto para las N ventas por vendedor.
  • Código limpio, comentado, sin errores de compilación.
🌉 Hacia las Funciones
Preparación para Unidad III
Has dominado todas las estructuras de control. Pero a medida que los programas crecen, un único main() con decenas de líneas se vuelve difícil de mantener. En la Unidad III aprenderás a dividir el código en funciones: bloques reutilizables con nombre, parámetros y valor de retorno que hacen cada parte más legible y fácil de probar.
