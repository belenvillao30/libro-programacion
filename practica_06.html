Sistema de Registro de Notas con Validación

while · do-while · validación de entrada · menú interactivo · estadísticas acumuladas

Duración180 min (3h)
ModalidadIndividual → Parejas
EntregableCódigo + bitácora
F1 · Modelado F2 · PSeInt F3 · C++ Individual F4 · Extensión F5 · Análisis IA F6 · Bitácora
Objetivo de la Sesión
El estudiante construye un sistema real de registro de notas usando while para acumulación y do-while para validación, combinando ambos ciclos con switch y estructuras selectivas en un menú interactivo completo.
🏫 Contexto del Problema
El departamento académico de ESPAM MFL necesita un sistema para registrar y analizar notas de estudiantes. El sistema debe: mostrar un menú principal con do-while, registrar notas de una materia con while hasta centinela, validar que cada nota esté en [0, 10] con un do-while interno, y calcular estadísticas al finalizar. Un registro incompleto (0 estudiantes) debe mostrar un mensaje apropiado.
F1
Modelado en Papel
0 – 20 min · SIN computadora

Antes de abrir el IDE, diseña el algoritmo completo en papel.

1
Diagrama de flujo del menú principal
Dibuja el diagrama del menú con opciones: (1) Registrar notas de una materia, (2) Ver reporte global, (3) Salir. El menú usa do-while. Para la opción 1, dibuja el ciclo interno de ingreso de notas con centinela (-1 para terminar) y el validador de rango.
2
Tabla de variables
Identifica todas las variables necesarias: ¿qué acumula? ¿qué compara? ¿cuáles son temporales y cuáles persistentes entre menús? Anota tipo de dato, nombre propuesto y valor inicial.
3
Simular con datos de prueba
Traza manualmente con notas: 8.5, 6.0, 9.5, -1 (centinela). Calcula a mano: total acumulado, cantidad, promedio, nota mínima y máxima. Estos serán tus valores esperados para verificar el programa.

Implementación en C++ — Individual
55 – 100 min

Traduce el pseudocódigo a C++. El código completo debe compilar sin errores y producir resultados idénticos a la simulación en papel (F1).

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    int    opcion, cantidad = 0;
    double nota, suma = 0, minNota, maxNota;

    do {
        // TODO: mostrar menú
        cin >> opcion;

        switch(opcion) {
            case 1:
                // TODO: leer notas con while y validar con do-while interno
                break;
            case 2:
                // TODO: mostrar reporte (verificar cantidad > 0)
                break;
            case 3:
                cout << "Hasta luego.\n";
                break;
            default:
                cout << "Opción inválida.\n";
        }
    } while (opcion != 3);

    return 0;
}

F4
Extensión en Parejas
100 – 135 min

Intercambia tu código con un compañero. Revísalo, comprende su lógica y luego entre los dos implementen las siguientes mejoras:

A
Clasificación de notas
Al mostrar el reporte, contar cuántas notas son: sobresaliente (≥9), satisfactorio (7–8.9), básico (5–6.9), insuficiente (<5). Mostrar conteo y porcentaje de cada categoría.
B
Opción para reiniciar registro
Agregar opción 4 al menú: "Nuevo registro" que reinicie suma, cantidad, minNota y maxNota. Pedir confirmación con do-while: "¿Está seguro? (s/n)". Validar que solo se acepte 's' o 'n'.
F5
Análisis Crítico con IA — Documentado
135 – 160 min
🤖 Protocolo de Análisis IA · Sesión 6

Pregunta a la IA: "Reescribe el ciclo principal de este sistema usando while(true) y break en lugar de do-while." Luego responde por escrito: (1) ¿Qué cambió estructuralmente? (2) ¿El comportamiento es exactamente igual? (3) ¿Qué versión te parece más legible y por qué? (4) ¿En qué situaciones while(true) sería preferible? Firma tu análisis.

📌 Regla de uso de IA en esta práctica

Puedes consultar a la IA para entender errores de compilación o conceptos. No puedes pedirle que escriba el código completo de las fases F3 o F4. Todo código entregado debe ser tuyo y debes poder explicar cada línea.
