A.1

Guía de Instalación de Herramientas

Todas las herramientas usadas en el curso son gratuitas y de código abierto. Instálalas en el orden indicado para evitar conflictos de configuración.

⚙️
MinGW-w64 (compilador g++)
Compilador GCC para Windows que permite ejecutar programas C++ desde la terminal. Es el compilador oficial del curso.
winlibs.com → descargar UCRT runtime, x86_64
📝
Visual Studio Code
Editor de código ligero con resaltado de sintaxis, terminal integrada y extensiones para C++. Extensión recomendada: C/C++ de Microsoft.
code.visualstudio.com → Download for Windows
🔷
PSeInt
Entorno para escribir y ejecutar pseudocódigo en español. Usado en las fases F2 de todas las prácticas para diseñar algoritmos antes de codificar en C++.
pseint.sourceforge.net → Descargar
🐱
Scratch 3
Entorno visual de programación por bloques. Usado en la Unidad I para introducir conceptos de secuencia, condición y repetición sin sintaxis formal.
scratch.mit.edu → Crear (no requiere instalación)
🌐
Compilador online (respaldo)
Para situaciones sin acceso a la instalación local. Soporta C++ estándar con entrada de datos.
onlinegdb.com/online_c++_compiler
🤖
Claude / ChatGPT (IA asistida)
Herramientas de IA para análisis crítico en la Fase 5 de cada práctica. Su uso requiere documentación y análisis propio. Ver protocolo en cada práctica.
claude.ai · chatgpt.com
Terminal Windows (CMD o PowerShell) · Verificar instalación 
# 1. Verificar que g++ está instalado y en el PATH
g++ --version
# Salida esperada: g++ (MinGW-W64...) 13.x.x

# 2. Compilar y ejecutar tu primer programa
g++ -Wall -Wextra -std=c++17 -o hola hola.cpp
./hola

# 3. Compilar con depuración (para encontrar errores)
g++ -Wall -Wextra -std=c++17 -g -o programa programa.cpp

# 4. Verificar versión de PSeInt (desde su interfaz gráfica)
# Menú Ayuda → Acerca de PSeInt
📌 Configuración recomendada de VS Code para C++

Instala la extensión C/C++ de Microsoft. Abre la paleta de comandos (Ctrl+Shift+P), busca C/C++: Edit Configurations y establece el compilador en la ruta de g++.exe de tu instalación MinGW. Activa IntelliSense para autocompletado y detección de errores en tiempo real. Para ejecutar con un atajo, configura una tarea de compilación en .vscode/tasks.json.

A.2

Cheat Sheet — Referencia Rápida de C++

Resumen de sintaxis esencial del curso. Úsalo como referencia rápida durante las prácticas y evaluaciones donde se permita material de apoyo.

Estructura básica de un programa 
#include <iostream>
#include <string>
#include <iomanip>
using namespace std;

int main() {
    // código aquí
    return 0;
}
Tipos de datos y declaración 
int    n = 42;          // entero
double x = 3.14;        // decimal
bool   ok = true;      // lógico
char   c = 'A';        // carácter
string s = "hola";     // cadena

cout << fixed << setprecision(2);
cin  >> n;
getline(cin, s);      // cadena con espacios
Estructuras selectivas 
if (cond) { ... }
else if (cond2) { ... }
else { ... }

switch (n) {
    case 1: ...; break;
    case 2: ...; break;
    default: ...;
}

// Ternario
int max = (a>b) ? a : b;
Estructuras repetitivas 
for (int i=0; i<n; i++) { ... }

while (condicion) { ... }

do {
    ...
} while (condicion);

break;    // salir del bucle
continue; // siguiente iteración
Funciones 
// Prototipo
double suma(double a, double b);

// Por valor · por referencia · const&
void f(int x,          // copia
        int& y,         // alias (modifica)
        const string& s); // solo lectura

// Recursión
int fact(int n) {
    if (n==0) return 1;
    return n * fact(n-1);
}
Arreglos y matrices 
int v[10];                  // arreglo
int m[3][4];               // matriz 3×4

// Recorrer arreglo
for (int i=0; i<10; i++)
    v[i] = i * 2;

// Pasar arreglo a función
void f(int arr[], int n);
void g(int m[][4], int fil);
Cadenas (string) s.length() // longitud s.substr(i, n) // subcadena s.find("hola") // posición s.at(i) // carácter en i s + " mundo" // concatenar s == "abc" // comparar toupper(c) // mayúscula tolower(c) // minúscula isdigit(c) // es dígito? isalpha(c) // es letra?
Formato de salida (iomanip) 
cout << fixed
     << setprecision(2)  // 2 decimales
     << setw(10)         // ancho campo
     << left              // alinear izq
     << right             // alinear der
     << setfill('0')     // relleno
     << 3.14159;        // → 3.14

to_string(42)  // int → string
stoi("42")    // string → int
stod("3.14")  // string → double
Búsqueda y ordenamiento // Búsqueda lineal for (int i=0; i<n; i++) if (v[i]==x) return i; // Búsqueda binaria (arreglo ordenado) int ini=0, fin=n-1; while(ini<=fin) { int mid=(ini+fin)/2; if(v[mid]==x) return mid; if(v[mid]<x) ini=mid+1; else fin=mid-1; } // Burbuja for(int i=0;i<n-1;i++) for(int j=0;j<n-i-1;j++) if(v[j]>v[j+1]) swap(v[j],v[j+1]);
📌 Operadores frecuentes

Aritméticos: + - * / % — Relacionales: == != < > <= >= — Lógicos: && || ! — Asignación: = += -= *= /= %= — Incremento: ++ -- — Módulo: a % b da el residuo de la división entera. División entera: 7 / 2 = 3 (ambos operandos enteros). Para decimales: 7.0 / 2 = 3.5.

A.3

Glosario de Términos del Curso

Unidad I — Fundamentos
Algoritmo
Secuencia finita, ordenada y sin ambigüedades de instrucciones que resuelve un problema o realiza una tarea.
Variable
Espacio con nombre en memoria que almacena un valor modificable durante la ejecución del programa.
Constante
Valor que no cambia durante la ejecución. En C++ se declara con const.
Tipo de dato
Clasificación que determina qué valores puede almacenar una variable y qué operaciones se pueden realizar: int, double, bool, char, string.
Pseudocódigo
Representación de un algoritmo en lenguaje cercano al natural, con estructuras formales de control, sin depender de la sintaxis de un lenguaje específico.
Compilador
Programa que traduce el código fuente (C++) a código máquina ejecutable. En el curso usamos g++.
Depuración
Proceso de identificar y corregir errores en un programa. Incluye errores de sintaxis (compilación) y errores de lógica (ejecución).
Error de sintaxis
Violación de las reglas gramaticales del lenguaje. El compilador lo detecta y el programa no compila.
Error de lógica
El programa compila y ejecuta, pero produce resultados incorrectos. Es el error más difícil de detectar.
Unidad II — Control
Estructura selectiva
Instrucción que ejecuta un bloque de código dependiendo de una condición. En C++: if-else, switch.
Estructura repetitiva
Instrucción que repite un bloque de código mientras se cumpla una condición. En C++: for, while, do-while.
Condición
Expresión que se evalúa como verdadera (true) o falsa (false). Controla el flujo de if y bucles.
Iteración
Cada ejecución individual del cuerpo de un bucle. Un bucle de 10 iteraciones ejecuta su cuerpo 10 veces.
Bucle infinito
Bucle cuya condición de salida nunca se cumple. Generalmente es un error de lógica, salvo en casos intencionales de espera.
Acumulador
Variable que suma valores sucesivos en un bucle. Se inicializa en 0 antes del bucle. Ejemplo: suma += v[i].
Unidad III — Funciones
Función
Bloque de código con nombre que realiza una tarea específica, puede recibir parámetros y devolver un valor.
Prototipo
Declaración anticipada de la firma de una función (tipo de retorno, nombre, parámetros) antes de su implementación.
Paso por valor
La función recibe una copia del argumento. Las modificaciones no afectan la variable original del llamador.
Paso por referencia (&)
La función recibe un alias directo de la variable original. Las modificaciones sí afectan la variable del llamador.
const&
Referencia de solo lectura. Evita copiar objetos grandes y garantiza que el original no sea modificado. Ideal para string de entrada.
Recursión
Técnica donde una función se llama a sí misma con argumentos que se acercan al caso base. Requiere caso base y caso recursivo.
Pila de ejecución
Estructura de memoria LIFO donde se apilan los marcos de cada llamada a función activa. El desbordamiento ocurre si es demasiado profunda.
Caso base
Condición en una función recursiva que detiene la recursión y devuelve un valor conocido sin hacer otra llamada.
Unidad IV — Arreglos
Arreglo
Colección de elementos del mismo tipo almacenados en posiciones contiguas de memoria, accedidos por un índice que empieza en 0.
Índice
Número entero que indica la posición de un elemento en un arreglo. El primer elemento tiene índice 0, el último n−1.
Arreglos paralelos
Varios arreglos donde el mismo índice i contiene datos relacionados del mismo "registro". El índice actúa como clave.
Matriz
Arreglo bidimensional de filas y columnas. Se accede con dos índices: m[fila][columna].
Búsqueda lineal
Recorre el arreglo elemento por elemento hasta encontrar el buscado. Costo O(n). No requiere orden previo.
Búsqueda binaria
Divide el arreglo ordenado a la mitad en cada paso. Costo O(log n). Requiere que el arreglo esté ordenado.
Ordenamiento burbuja
Compara pares adyacentes e intercambia si están fuera de orden. Costo O(n²). Simple pero ineficiente para n grandes.
Eliminación lógica
Marcar un registro como inactivo con un arreglo bool activo[] sin borrar físicamente los datos. Preserva la consistencia del arreglo.

📌 Política de integridad académica y uso de IA

El uso de inteligencia artificial en el curso es permitido y documentado, nunca oculto. El código generado por IA que se entregue sin análisis crítico se considera plagio académico. La documentación del protocolo de IA en la Fase 5 de cada práctica es evidencia de aprendizaje auténtico. En las evaluaciones presenciales calificadas, el uso de IA está explícitamente prohibido salvo indicación del docente.

🤖 Política de citación de IA en trabajos académicos

Cuando uses inteligencia artificial en un trabajo o práctica, debes citarla como herramienta: "Análisis asistido por Claude (Anthropic, 2025) / ChatGPT (OpenAI, 2025). Prompt utilizado: [texto del prompt]. Respuesta analizada y modificada por el autor." La IA es una herramienta de apoyo, no un autor. El estudiante es el único responsable del contenido que entrega.
