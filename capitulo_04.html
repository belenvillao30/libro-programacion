4.1

La Decisión como Estructura Algorítmica

Hasta ahora todos nuestros programas son secuenciales: ejecutan las instrucciones de arriba a abajo, siempre en el mismo orden. Pero los problemas reales requieren que el programa elija qué hacer según los datos.

Definición formal

Una estructura selectiva evalúa una condición booleana (verdadero o falso) y, según el resultado, ejecuta uno de dos caminos posibles. Es el mecanismo que convierte un programa en un sistema capaz de responder al contexto.

🤖 IA y las estructuras de decisión

La IA puede generar estructuras if-else correctas sintácticamente, pero la lógica de las condiciones depende de entender el problema. Un >= mal colocado, un && que debería ser ||, o condiciones en orden incorrecto producen comportamientos silenciosamente erróneos que solo tú puedes detectar.

⚠️ Error frecuente — comparar rangos

En C++ no se puede escribir 7 <= nota <= 10 como en matemáticas. El compilador lo acepta pero da resultado incorrecto. La forma correcta es: nota >= 7 && nota <= 10

4.3

if Simple y if-else

if simple — un solo camino

C++ · if simple
if_simple.cpp 
double nota = 8.5;

// Solo se ejecuta si la condición es verdadera
if (nota >= 7.0) {
    cout << "Aprobado" << endl;
}
// Si nota < 7.0, no imprime nada y continúa

if-else — dos caminos mutuamente excluyentes

C++ · if-else
if_else.cpp 
double nota;
cout << "Ingrese la nota: ";
cin  >> nota;

if (nota >= 7.0) {
    cout << "Estado: APROBADO" << endl;
} else {
    cout << "Estado: REPROBADO" << endl;
}
// Exactamente uno de los dos bloques se ejecuta siempre

📌 Llaves siempre, aunque sea una sola instrucción

C++ permite omitir las llaves si el bloque tiene una sola línea, pero siempre ponlas. Sin llaves, agregar una segunda instrucción al bloque crea un error lógico silencioso muy frecuente.

4.4

if-else if Encadenado — Múltiples condiciones

Cuando hay más de dos casos posibles se encadenan condiciones con else if. Solo se ejecuta el primer bloque cuya condición sea verdadera; los demás se saltan.

C++ · if-else if
calificacion.cpp 
#include <iostream>
using namespace std;

int main() {
    double nota;
    cout << "Nota (0-10): ";
    cin  >> nota;

    if      (nota >= 9.0) { cout << "Excelente";   }
    else if (nota >= 7.0) { cout << "Aprobado";    }
    else if (nota >= 5.0) { cout << "Recuperación"; }
    else                    { cout << "Reprobado";    }

    cout << endl;
    return 0;
}

📌 El orden de las condiciones importa

Si pones nota >= 5 antes que nota >= 7, una nota de 8.5 caería en "Recuperación". Las condiciones deben ir de mayor a menor (o de más restrictiva a menos restrictiva).

4.5

Condicionales Anidados y Operadores Lógicos

A veces la decisión depende de dos o más condiciones simultáneas. Hay dos formas equivalentes: anidar if dentro de otro if, o combinar condiciones con && y ||.

if anidado
if (edad >= 18) {
    if (tieneLicencia) {
        cout << "Puede conducir";
    } else {
        cout << "Sin licencia";
    }
} else {
    cout << "Menor de edad";
}
Con operador && (equivalente)
if (edad >= 18 && tieneLicencia) {
    cout << "Puede conducir";
} else if (edad >= 18) {
    cout << "Sin licencia";
} else {
    cout << "Menor de edad";
}

Ejemplo completo: clasificación de IMC con condiciones compuestas

C++
imc_clasificacion.cpp 
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double peso, estatura, imc;

    cout << "Peso (kg):     "; cin >> peso;
    cout << "Estatura (m):  "; cin >> estatura;

    imc = peso / (estatura * estatura);

    cout << fixed << setprecision(2);
    cout << "IMC: " << imc << endl;

    if      (imc < 18.5)                       { cout << "Bajo peso";       }
    else if (imc >= 18.5 && imc < 25.0)        { cout << "Normal";           }
    else if (imc >= 25.0 && imc < 30.0)        { cout << "Sobrepeso";        }
    else                                         { cout << "Obesidad";         }

    cout << endl;
    return 0;
}
4.6

switch-case: Decisión sobre Valores Discretos

switch compara una variable con valores enteros o caracteres exactos (no rangos). Es más legible que muchos else if cuando los casos son fijos y conocidos.

Con if-else if — verboso
if      (opcion == 1) { /* suma */   }
else if (opcion == 2) { /* resta */  }
else if (opcion == 3) { /* mult */   }
else if (opcion == 4) { /* div */    }
else { cout << "Inválido"; }
Con switch — limpio ✅
switch (opcion) {
    case 1: /* suma */  break;
    case 2: /* resta */ break;
    case 3: /* mult */  break;
    case 4: /* div */   break;
    default: cout << "Inválido";
}

Ejemplo completo: menú de calculadora

C++
calculadora_switch.cpp 
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double a, b;
    int    opcion;

    cout << "=== CALCULADORA ===\n";
    cout << "1. Suma   2. Resta   3. Multiplicar   4. Dividir\n";
    cout << "Opción: "; cin >> opcion;
    cout << "Num 1: ";  cin >> a;
    cout << "Num 2: ";  cin >> b;

    cout << fixed << setprecision(2) << "Resultado: ";

    switch (opcion) {
        case 1:
            cout << a + b;
            break;
        case 2:
            cout << a - b;
            break;
        case 3:
            cout << a * b;
            break;
        case 4:
            if (b != 0)
                cout << a / b;
            else
                cout << "Error: división por cero";
            break;
        default:
            cout << "Opción no válida";
    }
    cout << endl;
    return 0;
}
⚠️ El break es obligatorio

Sin break, la ejecución continúa al siguiente case automáticamente (fall-through). Esto es casi siempre un error. La única excepción intencional es cuando varios casos comparten el mismo bloque: case 1: case 2: /* código */ break;

Ejercicios de Aplicación

✏️ Actividades de Aplicación
1
Comprensión
Predicción de salida. Sin ejecutar, determina la salida para cada valor de x = 15: (a) if (x > 10 && x < 20) (b) if (x == 15 || x == 20) (c) if (!(x < 10)) (d) if (x >= 10 && x <= 10). Justifica cada respuesta con la tabla de verdad.
2
Aplicación
Sistema de calificación ESPAM. Escribe un programa que lea una nota (0–10) y muestre: Excelente (9–10), Muy Bueno (8–8.9), Bueno (7–7.9), Recuperación (5–6.9), Reprobado (<5). Incluye una condición que valide que la nota esté en el rango válido. Primero: pseudocódigo + tabla de simulación. Luego: código C++.
3
Aplicación
Calculadora con switch. Implementa una calculadora que lea dos números y un operador (+ - * /) ingresado como char. Usa switch sobre el carácter. Maneja la división por cero como caso especial dentro del case '/'.
4
Aplicación
Descuento por categoría. Un comercio aplica descuentos según el monto: <$50 → 0%, $50–$99 → 5%, $100–$199 → 10%, ≥$200 → 15%. Si además es cliente VIP (ingresar como bool), se suma un 3% adicional. Muestra monto original, porcentaje de descuento, descuento y total a pagar.
5
Integrador + IA
Auditar condición. Pide a la IA que genere un programa que determine si un año es bisiesto. Un año es bisiesto si: (divisible por 4 Y no divisible por 100) O (divisible por 400). Audita la condición generada: ¿usó correctamente &&, || y paréntesis? Verifica con los años 2000 (bisiesto), 1900 (no bisiesto) y 2024 (bisiesto).
Actividad Presencial Evaluativa — Capítulo 4
Sesión 4 · Tiempo: 45 min · Sin acceso a IA
Un banco aplica tasas de interés según el plazo de un depósito: 1 mes → 3%, 3 meses → 5%, 6 meses → 7%, 12 meses → 9%, otro plazo → inválido. El programa solicita monto y plazo (en meses), calcula el interés y muestra monto original, tasa aplicada, interés generado y monto final. Usa switch para el plazo y if-else para validar que el monto sea positivo.
  • Pseudocódigo correcto entregado en papel antes de codificar.
  • switch con los 4 casos válidos y default para plazo inválido.
  • Validación de monto positivo con if-else.
  • Salida formateada con fixed + setprecision(2).
  • Tabla de simulación con al menos 3 casos de prueba.
